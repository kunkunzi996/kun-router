# 13 Project Cleanup：平台与记忆边界

## 一句话定位

本文件只在项目洁癖发现 Agent 记忆、项目规则或平台路径问题时读取。

它负责区分“当前项目可处理的内容”“只能报告的内容”和“必须等用户确认的内容”。

## 目录

1. 通用判定
2. Claude Code
3. OpenAI Codex
4. 其他 Agent
5. PowerShell 与 POSIX 体检命令

## 通用判定

先确认文件归属，再读取或修改。

项目级内容必须同时满足：路径或平台映射能指向当前项目，并且内容没有明显属于其他项目的迹象。

归属不明时，整层标为“不适用”，不扩大扫描范围。

用户级全局规则不是项目规则层；机器生成状态不是项目文档层。

| 对象 | 默认动作 |
|---|---|
| 当前项目文档与项目规则 | 读取、依据证据修正可直接修的事实 |
| 可明确归属的项目级记忆 | 读取；修正有事实证据的内容 |
| 全局规则 | 只读取上级约束，修改前必须确认 |
| 机器生成记忆或状态 | 只检查和报告，不手工编辑 |
| 归属不明内容 | 不扫描、不编辑，报告“不适用” |

删除、缩短、移动或重命名任何记忆文件或段落，都必须待用户拍板。

## Claude Code

只有在 `~/.claude/projects/<project>/memory/` 能可靠映射到当前项目时，才把它视为项目级记忆。

`MEMORY.md` 的 200 行 / 25KB 是 Claude Code 的硬读取限制。

建议在约 70% 时预警，先识别可毕业的稳定知识；不要自动删除或压缩原内容。

可以直接修正有代码、配置、测试或当前文档证据支持的项目事实。

删除记忆文件、删除段落或缩成一行指针，必须列入待拍板。

`~/.claude/CLAUDE.md` 是用户级全局规则，不因单个项目洁癖自动修改。

项目内 `CLAUDE.md` 和 `.claude/rules/` 才属于项目规则层。

## OpenAI Codex

项目内 `AGENTS.md` 属于项目 AI 规则层。

`~/.codex/AGENTS.md` 是用户级全局指令，不属于当前项目记忆。

`~/.codex/memories/` 是 Codex 的机器生成状态，默认只检查和报告，不手工编辑。

不要把 Codex 机器记忆复制进项目，也不要把项目文档写回机器记忆目录。

若无法可靠区分记忆是否属于当前项目，跳过该层并在报告中写“不适用”。

优先用项目 `AGENTS.md`、项目文档和 Codex 的正式记忆控制入口保存长期项目知识。

## 其他 Agent

V0.7.8 不为未经本轮确认的平台写死专用路径。

只允许：

1. 探测项目内是否存在明确的规则或记忆目录。
2. 能确认归属和编辑规则时，按本门通用原则处理。
3. 无法确认时，跳过记忆层并明确说明。

不能因为目录名称看起来像“memory”就读取其他项目内容。

## PowerShell 与 POSIX 体检命令

以下命令只用于读取和记录尺寸，不会修改文件。

### PowerShell

```powershell
Get-Item -LiteralPath <文件> | Select-Object Length
(Get-Content -LiteralPath <文件>).Count
Resolve-Path -LiteralPath <路径>
Get-ChildItem -LiteralPath <目录> -File -Recurse
```

### POSIX Shell

```bash
wc -l <file>
wc -c <file>
realpath <path>
find <directory> -type f
```

命令输出只是现实证据的一部分。

发现尺寸接近阈值时，先报告哪一类内容导致膨胀，再决定是否需要用户拍板的清理动作。

## 最重要原则

先辨认归属，再处理内容。

项目洁癖绝不能借“整理记忆”为理由修改全局规则、机器状态或其他项目。
